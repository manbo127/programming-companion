# 程序设计学习智能学伴“ 小码”需求与约束分析

## 1. 项目目标

系统面向程序设计初学者，解决报错难理解、解题过程缺少引导、学习记录分散和反复受挫等问题。系统定位是“学习陪伴者”而非代写工具：帮助学习者理解原因、形成思路、完成验证，不直接交付整题答案。

## 2. 角色与使用方式

- 匿名学习者：无需注册即可创建对话、保存本设备身份和使用全部学习能力。
- 注册学习者：把当前匿名历史升级为账号数据，可在其他设备登录继续学习。
- 运维人员：负责环境变量、数据库迁移、备份、日志、指标和服务更新。
- 外部服务：DeepSeek API 只负责生成自然语言教学回答，不直接访问本地数据库。

## 3. 十项功能需求

### 3.1 基础对话

支持文本、代码和完整报错三类输入；自动识别报错解读、解题引导、知识问答和自由交流四种场景，也允许用户手动选择模式。

### 3.2 代码错误解读

识别错误类型、行列位置和语言，输出错误类别、含义与首项检查动作。回答必须以证据定位和理解原因为主，不重写完整程序。

### 3.3 解题思路引导

按照“理解题意—拆分问题—形成算法—验证与反思”逐轮推进；每次只推进一个认知步骤，并以可回答的小问题结束。

### 3.4 激励与情绪反馈

区分中性、困惑、挫败和积极状态，保存分数和连续趋势。用户可选择温暖、平衡或简洁反馈风格，系统应避免机械重复和普通问号误判。

### 3.5 Web 交互界面

首页保持单一对话入口；历史栏可开合；会话支持新建、重命名、删除；代码舱可独立开合；移动端可用；回答下方可显示结构化诊断和官方资料引用。

### 3.6 对话上下文

按 Token 预算选取最近消息，超出预算时用只读摘要衔接较早内容，避免简单按固定字符截断导致上下文断裂。

### 3.7 用户画像与跨对话记忆

保存昵称、学习阶段、常用语言、目标和反馈风格；仅从已完成消息的场景、语言、主题和错误类型生成结构化长期摘要，不把原始用户文本提升为系统指令。用户可查看、关闭、清除和重建记忆。

### 3.8 多语言代码支持

支持 Python、Java、C、C++、JavaScript、TypeScript、Go、Rust 和 SQL 的规则识别、语言提示与常见错误提取。

### 3.9 主动学习提醒

记录学习事件并按主题生成 1/3/7/14/30 天复习计划；到期计划只生成一次提醒，完成后推进下一间隔。

### 3.10 知识点问答增强

从本地可审计知识库检索相关摘要，回答同时返回真实官方文档链接。运行时不抓取网页，不把网页内容当作行为指令。

## 4. 工程与非功能需求

- 安全：密钥只来自环境变量；Cookie 签名、HttpOnly、SameSite，生产启用 Secure；写请求启用 CSRF；账号密码哈希；失败登录锁定；所有资源按 client_id 校验所有权。
- 可靠性：DeepSeek 调用设置单次与总超时，只重试限流、超时、连接和服务端错误；认证或请求参数错误不盲目重试。
- 数据一致性：SQLite 开启外键、WAL、busy timeout；所有结构变化通过单头 Alembic 迁移；备份后执行完整性检查。
- 可扩展性：Repository/Service/API 分层；数据库 URL 可切换 PostgreSQL；LLM 通过统一 Gateway 隔离。
- 可观测性：请求 ID、JSON 日志、健康检查、就绪检查、聚合指标和部署后冒烟测试。
- 可测试性：真实 API 不参与自动化测试；FakeLLM 覆盖对话链路；单元、集成和生产安全测试均可离线运行。
- 性能：单机 SQLite 使用一个 Gunicorn worker 和多线程；切换 PostgreSQL 后可扩展多个 worker 或多实例。
- 隐私：指标和日志不记录用户消息、代码、Cookie、密钥与数据库路径；知识引用与画像摘要均可解释。

## 5. 技术与部署约束

- 后端采用 Flask、SQLAlchemy 和 Alembic；前端采用原生 HTML/CSS/JavaScript。
- 默认使用 SQLite，课程部署不依赖额外数据库服务；保留 PostgreSQL 连接能力。
- 模型使用 DeepSeek OpenAI 兼容接口，默认 `deepseek-v4-flash`。
- 实际生产架构为阿里云香港 ECS + Nginx + Gunicorn + Systemd + HTTPS。
- 不采用 Vercel 作为当前生产运行环境，因为其 Python Function 临时文件系统不能保证 SQLite 长期持久化。
