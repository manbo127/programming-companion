1 引言

1.1 研究背景

编程能力已成为信息时代的重要基础素养，越来越多高校将程序设计类课程纳入通识教育。然而，编程学习门槛较高，初学者普遍面临以下困境。

第一，遇到问题无人可问。编写代码时一个简单的语法或逻辑错误就可能导致程序无法运行。编译器给出的英文报错信息晦涩难懂，自行排查效率极低。课堂之外，学生身边往往缺乏能即时解答疑惑的人。

第二，错误信息难以理解。即便是 Python 这样以简洁著称的语言，其运行时错误信息对初学者仍像天书。学生往往能读懂每个单词，却不理解错误的真正原因，更不知从何改起。

第三，网络资源大多呈现完整答案。当学生上网搜索求助时，找到的大多是现成的完整代码。这虽解决了当下问题，却绕过了思考的过程，不利于编程思维的形成。学生真正需要的是解题思路的引导者，而非答案的搬运工。

第四，缺乏持续学习动力。编程学习是反复试错的过程。初学者在连续遭遇报错后易产生强烈挫败感，甚至放弃。学习过程中缺少一个能及时给予鼓励、帮助建立信心的角色。

在此背景下，大型语言模型技术的快速发展为编程教育提供了新的可能。以 DeepSeek 为代表的国产大模型在中文理解与代码分析方面表现优异，且提供与 OpenAI 兼容的 API 接口，使得开发者能以较低成本构建智能教育工具。

1.2 研究目标

本项目旨在设计并实现一款基于大语言模型、面向编程初学者的 Web 对话式学习助手，即程序设计学习智能学伴。具体目标如下。

第一，打造耐心温和的编程学习陪伴者。学伴应具备自然流畅的对话能力，以鼓励式、启发式的语气与用户交流，而非机械地输出答案。角色定位为陪伴者而非教师，强调平等的学习伙伴关系。

第二，实现代码错误的通俗化解读。当用户提交出错代码和报错信息时，学伴能分析错误原因，用通俗语言解释为什么错和往哪个方向改。核心约束是不直接输出完整正确代码，确保用户仍需自己动手修改，从而真正理解错误。

第三，提供苏格拉底式的解题引导。面对解题求助，学伴采用分步引导方式，先帮助理解问题，再分解解题步骤，通过提问推动用户思考。每轮对话只推进一个步骤，让用户在思考中进步。

第四，融入情感化激励能力。学伴能感知用户学习状态，在连续成功时给予真诚的表扬，在受挫时给予及时的鼓励，通过情感化反馈帮助用户保持学习动力。

第五，构建友好易用的 Web 交互界面。提供专用代码输入区和错误信息输入区，以聊天气泡展示对话历史，界面设计简洁直观，降低使用门槛。

第六，具备对话上下文记忆能力。学伴能记住本次对话的交流历史，在后续回答中正确引用上下文，理解代词指代等语言现象，使多轮对话自然连贯。

概括而言，本项目的核心设计理念是引而不发，启而不代。引导用户思考，但不代替用户思考；启发用户寻找答案，但不直接提供答案。
